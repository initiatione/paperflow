The Paper Source skill bundle contract now fails when any Python script under `plugins/paper-source/scripts` exceeds 2000 lines. - Paper Source 2.3.9: finish the `promote-to-wiki` retirement across active surfaces and give `discover-to-handoff` a single route owner. Removes the retired `promote-to-wiki`/`promote-after-approval` readiness and recommended-action branches from `run_index.py` (including the `_promote_action_command` and `_paper_gate_allows_promotion` helpers), the `promote-to-wiki` entry from the batch awaiting-promotion recognition in `orchestrator.py`, the legacy stage line in `routing-rules.example.yaml`, and the `promote-to-wiki` fallback wording from the quality-gates metric pack and its test fixture. `discover-to-handoff` now triggers only the `paper_ingest` route instead of duplicating across `paper_discovery` and `paper_ingest`, with a routing-contract test guarding the single owner. Recommended next actions now recognize only the agent-mediated `wiki-ingest-handoff` -> `record-human-approval` -> `wiki-ingest-trigger` -> `record-wiki-ingest` flow; historical changelog mentions of the retired command remain as history. Source-checkout change only; marketplace refresh / reinstall / installed-cache verification remain to be run separately.
- Paper Source 2.3.8: make discovery ranking gates cross-discipline instead of tuned to one task/domain. `rank_candidates` now separates strict `priority_keywords` topic fit from long-profile positive keyword matching; when no hard anchors are supplied, positive keyword topic fit saturates after a small number of strong matches and preserves `keyword_coverage_score` only as diagnostics, so long multi-discipline profiles do not dilute a clearly relevant paper into `weak_topic_fit`. `quality_gate.dimensions` now exposes identity, relevance, inspectability, validation, source confidence, reproducibility, and request risk for every ranked candidate. Docs now clarify the `openags/paper-search-mcp` boundary: it is a broad retrieval, metadata, dedupe, download, and source-coverage provider, not a semantic quality gate or recommendation ranker; Paper Source owns query planning, source routing, quality gates, DOI policy, and chat-facing recommendation surfaces. Source-checkout change only; marketplace refresh / reinstall / installed-cache verification remain to be run separately.
- Paper Source 2.3.7: calibrate discovery quality gates and recommendation status taxonomy after live AUV evidence showed clearly in-domain DOI/PDF papers being marked `quality_tier=Reject` only because broad query/profile keywords diluted hard-anchor topic fit. Query-plan priority ranking now uses hard domain anchors/domain focus terms instead of method/problem/context filler as the topic-fit denominator, while those soft terms remain ranking evidence. `session_recommendations.review_appendix` is now strictly non-Reject; quality Reject records move to `quality_reject_debug` diagnostics with blocking reasons and query/source provenance. Zero-primary runs now expose `no_primary_recommendations_summary` in `report.json` and `report.md` so agents can explain whether DOI policy, quality gates, existing-library saturation, or recall gaps caused an empty primary list. Source-checkout change only; marketplace refresh / reinstall / installed-cache verification remain to be run separately.
- Paper Source 2.3.6: tighten the chat-facing existing-library format after AUV discovery feedback and make `_meta/reference-index.json` the canonical dedupe backlog when available. `paper-discovery` output docs now require `库中已有，可回看` to be rendered as a short explanatory sentence plus a Markdown table with paper/year/status/entry/DOI columns, instead of a one-line semicolon-separated list. `report.md` now mirrors this table shape under `Already In Library Or Wiki`, preserving the boundary that already-in-wiki/raw-library hits are reminders, not new recommendations. `paper_library.py` now skips raw metadata scans when wiki `_meta/reference-index.json` loads successfully and reports `raw_scan_policy`; `_paper_source/raw/*/metadata.json` is only a missing/unreadable-index fallback. Source-checkout change only; marketplace refresh / reinstall / installed-cache verification remain to be run separately.
- Paper Source 2.3.5: require DOI-present chat-facing discovery recommendations without silently dropping promising papers. `normalize_candidates` now enriches DOI from provider DOI fields, DOI URLs, raw metadata, and deterministic arXiv DOI `10.48550/arXiv.<base_id>`, extracts public repository URLs from provider abstracts/snippets, and records enrichment provenance. Dry-run can run targeted Grok/web DOI recovery for remaining DOI-missing candidates, writing `doi-recovery-grok-*` artifacts before re-normalizing. `session_recommendations.primary_recommendations` and `review_appendix` now exclude remaining DOI-missing candidates and report recovery success/failure through `doi_recovery_summary`, `doi_resolution_summary`, and `doi_filtered_summary` / `missing_required_doi`; output docs now require the agent to keep primary recommendations, `新增待复核候选`, and `库中已有，可回看` separate and to report Grok as valid evidence only when the provider status is truly `ok` with usable records and no fallback warnings. Source-checkout change only; marketplace refresh / reinstall / installed-cache verification remain to be run separately.
- Paper Source 2.3.4: make optional Grok supplemental discovery more resilient for configured OpenAI-compatible runtimes. The adapter now preserves the user's primary `OPENAI_COMPATIBLE_MODEL`, retries configurable/default multi-agent-first model fallbacks for `grok_provider_error` or `source_fallback` MCP responses, records non-secret `model_attempts` in `grok-search-raw.json`, and makes parallel Grok fan-in grace configurable through `PAPER_SOURCE_GROK_PARALLEL_GRACE_SECONDS` with a 180 second default. Docs now clarify that each user must configure their own Grok relay endpoint/key/model and that local proxy, TUN/fake-IP, DNS, or Cloudflare-challenge symptoms are runtime diagnostics, not plugin defaults. Source-checkout change only; marketplace refresh / reinstall / installed-cache verification remain to be run separately.
