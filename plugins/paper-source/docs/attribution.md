# EPI Attribution

Discovery wraps the external `openags/paper-search-mcp` MCP server and `paper-search` CLI fallback, then records command, source, query, source coverage, and raw-response provenance in each run. `prepare-ranked` is the narrow search -> acquire -> MinerU parse path for raw preparation only. Its acquisition path tries MCP `download_with_fallback` first, records `fallback_chain`, `use_scihub`, MCP probe, and upstream tool provenance in `acquire-record.json` when available, and only continues to source-native MCP tools, CLI download, or direct URL fallback when there is a direct PDF candidate to try. If no direct PDF URL exists and OA fallback finds no PDF, acquisition records `failure_class=manual-download-required` plus `manual_download.candidate_manual_urls` for user-side organization/institution download. After successful acquisition, EPI may call MCP `read_<source>_paper` or CLI read to write the non-authoritative retrieval preview sidecar `paper-search-read-preview.txt` and record `retrieval_preview`; this preview helps inspect upstream retrieval but is not the source of final wiki evidence and is not replacing MinerU. MinerU parsing is optional and uses the local `skills\mineru-paper-parser\scripts\mineru_batch_to_md.py` entrypoint; successful runs keep final parse artifacts under `mineru/` and only lightweight command logs under `mineru-command/`.

MinerU parsing is an internal helper capability for EPI, not a separate marketplace plugin.

EasyScholar quality enrichment ports the API integration pattern from `chaosman42/easyscholar-mcp`, an MIT License project, into EPI's native Python discovery pipeline. EPI reuses the EasyScholar publication-rank endpoint semantics and `EASYSCHOLAR_SECRET_KEY` environment convention, but does not embed the upstream Node MCP server. The enrichment writes `easyscholar-record.json` and candidate `verified_metrics.easyscholar` evidence when available; missing keys or API failures are soft failures and reported as `未核实`.

EPI keeps raw artifacts under `_epi/raw` and evidence handoff bundles under `_epi/staging`. Final Obsidian/LLM Wiki pages are written by the wiki ingest agent according to the target vault contract; the formal EPI page families are `references/`, `concepts/`, `derivations/`, `experiments/`, `synthesis/`, `reports/`, and `opportunities/`. EPI can hand off evidence for these families, but it does not decide final page authority.
